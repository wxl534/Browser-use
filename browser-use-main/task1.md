# 大英图书馆图片下载任务配置

## 任务目标
访问大英图书馆网站，通过 International Dunhuang Programme (IDP) 搜索并下载与中国佛教相关的图片到本地目录，自动识别图片下载链接并下载，使用网页中的图片标题命名

## 执行流程概览
1. **访问网站**：打开大英图书馆官网，导航到 IDP 数字收藏页面
2. **搜索关键词**：使用中国佛教相关关键词搜索（如 china temple, china buddhist, china pagoda, monk 等）
3. **提取 title 与介绍**：识别网站的 DOM 结构，提取每张图片的标题信息和介绍
4. **下载图片**：自动识别图片的下载链接并下载到指定目录
5. **保存 title**：在运行结束前，整理所有 title 和介绍信息，按顺序列出

## 注意事项
不要太过频繁连续地访问网站，避免被封禁，遇到卡住的部分，稍等 30 秒后重新执行

## 性能优化与反检测策略（针对新大模型）

### 1. 降低访问频率
- **搜索间隔**：每次搜索后等待 60 秒再执行下一次操作
- **页面停留**：每个页面至少停留 10-15 秒，模拟真实阅读时间
- **下载间隔**：每下载 1 张图片后等待 30 秒再处理下一张
- **关键词切换**：完成一个关键词的所有图片后，等待 120 秒再尝试下一个关键词

### 2. 增加步骤间等待时间
- **点击后等待**：每次点击操作后等待 5-8 秒
- **页面加载**：页面跳转后等待 10-15 秒确保完全加载
- **下载确认**：点击下载按钮后等待 30-45 秒确认下载完成
- **返回操作**：使用返回按钮后等待 8-10 秒

### 3. 减少并发请求
- **单线程处理**：一次只处理一张图片，完成整个流程后再处理下一张
- **禁止批量操作**：不要同时打开多个详情页或下载多个文件
- **顺序执行**：严格按照"点击→提取→下载→验证→返回"的顺序，每步完成后才继续

### 4. 模拟真实用户行为
- **随机延迟**：在固定等待时间基础上添加 2-5 秒的随机延迟
- **分步操作**：将复杂操作分解为多个小步骤，每步之间有自然间隔
- **滚动页面**：在搜索前缓慢滚动页面 2-3 次，每次间隔 3 秒
- **鼠标移动**：点击前先移动到元素位置，停留 1-2 秒再点击
- **视线轨迹**：按照"搜索框→搜索按钮→结果区域"的自然顺序移动

### 5. LLM 提示词优化
在任务描述中明确强调：
```
重要：请像一个真实的人类用户那样操作：
- 动作不要太快，要有合理的停顿和思考时间
- 每次点击前稍微停顿一下
- 页面加载时耐心等待
- 不要表现出机器人才有的快速连续操作
- 如果感觉到卡顿是正常的，不要重复操作
```

## 详细执行步骤

### 阶段 1 - 网站导航与搜索
1. **访问大英图书馆官网**
   - URL: https://www.bl.uk/
   
2. **导航到 Collection 页面**
   - 点击网页顶部的"Collection"菜单
   
3. **找到 IDP 数字收藏**
   - 在"Digital collections"部分找到"International Dunhuang Programme (IDP)"
   - 点击进入 IDP 专题页面
   
4. **使用关键词搜索**
   - 在搜索框输入中国佛教相关关键词（按优先级）：
     - china temple
   - 输入完成后等待 5 秒再点击搜索按钮
   - 点击"Search"按钮提交搜索
   - 等待 15 秒让搜索结果完全加载
   
5. **浏览搜索结果**
   - 查看搜索结果列表
   - 确认结果包含相关图片，只关注前n=1张图片

### 阶段 2 - 图片信息提取与下载（核心功能）
**对搜索结果中的每张图片执行以下操作：**

1. **点击图片进入详情页**
   - 在搜索结果中缓慢移动鼠标到第一张图片上
   - 停留 2 秒后点击
   - 等待页面加载完成（至少 15 秒）

2. **提取图片信息**
   - 图片标题（Title）
   - 图片介绍/描述（Description/Notes）
   - 索引编号（1-n）

3. **识别并点击下载**
   - 在详情页找到"Download center image"或下载按钮
   - 观察按钮位置 3 秒后点击
   - 每个图片只用点击一次按钮
   - 设置下载目录为：`D:\desktop\browser-use-main\image`
   - 点击下载按钮
   - 在确认对话框中选择"确定"完成下载
   - 等待 30-45 秒确保下载完成
   
4. **验证下载成功**
   - 检查文件是否存在于下载目录
   - 确认文件大小不为 0
   - 如果失败，等待 30 秒后重试（最多 2 次）

5. **返回搜索结果**
   - 使用浏览器返回按钮回到搜索结果页
   - 点击返回后等待 10 秒
   - 页面稳定后休息 30 秒再处理下一张图片
   - 重复以上步骤

**重复以上步骤直到下载完所有相关图片**

### 阶段 3 - 展示 title 与图片信息
1. **所有图片处理完成后，统一展示信息**
   - 将所有识别到的 title 和介绍信息按顺序编号（1.、2.、...、n）
   
2. **输出格式要求**：
   ```
   Title.txt:
   [标题 1]
   [标题 2]
   ...
   [标题 n]
   END
   
   Info.txt:
   [介绍 1]
   [介绍 2]
   ...
   [介绍 n]
   END
   ```
   
3. **注意事项**：
   - 必须换行输出，不能使用 `\n` 字符
   - title 和介绍顺序必须与下载顺序严格对应
   - 每个关键词搜索的结果单独记录

### 阶段 4 - 多关键词搜索（可选）
1. **完成当前关键词后**
   - 返回搜索框
   - 清除之前的关键词
   
2. **输入下一个关键词**
   - 按照优先级列表尝试其他关键词
   - 重复阶段 2-3 的流程
   
3. **避免重复下载**
   - 检查已下载的文件名
   - 跳过已经下载过的图片

### 阶段 5 - 运行重命名脚本（离线处理）
**注意：此阶段在 browser-use 任务完成后由 IDE 自动执行**

1. **执行重命名脚本**
   ```bash
   python rename_images.py
   ```

2. **脚本自动完成以下工作**：
   - 从 `D:\desktop\browser-use-main\browseruse_agent_data\title.txt` 读取所有 title
   - 从 `D:\desktop\browser-use-main\browseruse_agent_data\info.txt` 读取所有介绍信息
   - 扫描 `D:\desktop\browser-use-main\image` 目录中的所有图片文件（按修改时间排序）
   - 将 title 按顺序分配给图片
   - 重命名图片文件为对应的 title（清理非法字符）
   - 为每个图片创建对应的 .txt 信息文件，内容为介绍信息
   - 生成重命名记录文件 `rename_record.txt`

3. **最终输出**：
   - 重命名后的图片文件（如 `Buddhist_temple_Chelmsford.tiff`）
   - 对应的信息文件（如 `Buddhist_temple_Chelmsford.txt`）
   - 重命名记录文件（包含原文件名和新文件名映射）

## 工具与脚本说明

### browser-use 内置功能
- **文件下载**：自动识别下载链接或使用图片 URL 下载
- **JavaScript 执行**：用于分析 DOM 并提取图片 title 和介绍
- **文件写入**：使用 write_file 动作将 title 列表保存到 title.txt
- **页面导航**：click、go_to_url、navigate_back 等操作
- **等待机制**：等待下载完成、页面加载

### rename_images.py 脚本
- **功能**: 离线重命名已下载的图片文件并创建信息文件
- **输入**: 
  - `title.txt`（title 列表）
  - `info.txt`（介绍信息列表）
  - `image` 目录（已下载的图片）
- **输出**: 
  - 重命名后的图片文件
  - 对应的 .txt 信息文件
  - `rename_record.txt`（重命名记录）
- **使用方法**: `python rename_images.py`

## 错误处理

### 下载失败
- 等待时间：30 秒（等待下载完成）
- 重试次数：最多 2 次
- 仍失败则：记录 `SKIP-{index}: 下载失败` 并继续下一个

### 标题提取失败
- 如果无法找到标题，使用 URL 中的文件名作为替代
- 在 title.txt 中记录为占位符（如 `image_1`, `image_2`）

### 介绍信息提取失败
- 如果无法找到介绍，记录为 "No description available"
- 在 info.txt 中记录默认值

### 重命名失败
- 如果 title.txt 不存在，脚本会提示错误并退出
- 如果图片数量与 title 数量不匹配，只处理较小数量的那一方
- 如果文件名冲突，自动添加序号后缀（如 `title_1.png`, `title_2.png`）
